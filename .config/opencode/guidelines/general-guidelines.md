# Coding Style

## Naming

- Use descriptive names that reflect the purpose of the code
- Constants should be in UPPER_SNAKE_CASE only for global constants

## Functions
- A function should do one thing
- Prefer fewer arguments
- Avoid side effects within a function unless it is necessary

## Files
- Try to keep files short
- Always use typing within a file

## Comments
- Leave little to no comments within the code you write
- If you add comments while coding remove these comments out before returning the final product
- The code should be able to be read and not need any comments

## Dependencies
- Prefer standard library dependencies over external dependencies

## Tests
- Keep test cases within files along side where the code actually lives
